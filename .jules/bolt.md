## 2024-03-05 - [Scroll Event Optimization]
**Learning:** This Next.js application requires scroll event listeners to be heavily optimized to prevent main thread blocking, specifically because it uses a lot of visual effects. Binding `setOffsetY(window.scrollY)` directly to the `scroll` event can cause layout thrashing and synchronous re-renders, hurting the user experience, specially in components like `AnimatedBannerBackground` or `PageHeader`.
**Action:** Always wrap state updates in scroll event listeners with `window.requestAnimationFrame()` alongside a `ticking` boolean flag to throttle the updates. Additionally, the `scroll` event listener should have `{ passive: true }` attached so the browser knows the event won't `preventDefault()`, allowing for smoother scrolling.
